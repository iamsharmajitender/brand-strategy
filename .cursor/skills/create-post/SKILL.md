# Create LinkedIn Voice Post

Write LinkedIn posts that match the published voice templates (POV, LRN, ARC, EXP). **One post per Insight**, same tags, shorter format, link back to site.

**Template:** [template.md](template.md) — read before drafting.

**Source docs:** channel rules `docs/100 post/10 Overview.mdx` · voices `docs/100 post/20 POV.mdx`, `30 LRN.mdx`, `40 ARC.mdx`, `50 EXP.mdx` · worked example `docs/100 post/article/10 Example-POV-Hallucinations.mdx` · tags `docs/00 content-bank/50 Tagging-Guidelines.mdx`.

## The one rule

**One Insight to one LinkedIn voice post.** Not a separate backlog. Not multiple posts per Insight. The post uses the **same voice and same tags** as its parent Insight, extracts one takeaway from THE CLAIM, and links back.

Post Engine to LinkedIn. Insight Engine to Insights on site. Never swap them.

## The four voices

Match the **parent Insight's primary voice** — do not pick a different voice for LinkedIn.

| Slug | Voice | Core question | Length |
| --- | --- | --- | --- |
| `pov` | **POV** | What belief am I challenging? | 150-400 words |
| `lrn` | **LRN** | What did I learn in the field? | 200-500 words |
| `arch` | **ARC** | What is the architecture pattern or flow? | 300-600 words |
| `exp` | **EXP** | How do I explain this without losing truth? | 200-400 words |

## Workflow

1. **Confirm the parent Insight is published** — voice and tags locked.
2. **Match the voice** — same as the Insight (`pov` / `lrn` / `arch` / `exp`).
3. **Copy the tags** — identical Core pillar + voice + topic tags as the parent.
4. **Extract one takeaway** — one idea from THE CLAIM, not the whole Insight.
5. **Draft in the voice structure** — see per-voice structures in `template.md`.
6. **Add the CTA** — link back to the parent Insight on site.
7. **Validate** — run the per-voice checklist in `template.md`.

## Per-voice structure (summary)

Full templates and worked examples are in `template.md`.

- **POV** — Hook (strong belief) → sharp statement → why it is misunderstood → what actually matters → enterprise consequence → memorable closing principle.
- **LRN** — Situation → what broke/surprised you → 3-5 concrete learnings → reframed takeaway.
- **ARC** — Problem statement → architecture flow → layer responsibilities → one design principle.
- **EXP** — What people get wrong → the simple truth → mental model → what you must not lose → one practical takeaway.

## Required elements

Every post must include:

- **Same voice** as the parent Insight
- **Same tags** as the parent Insight (Core pillar + voice + topics)
- **One takeaway** extracted from THE CLAIM (not a mini-essay)
- **A memorable closing line** the reader can repeat
- **Link back** to the parent Insight on site

## Voice rules

- **One idea, not a summary** — extract a single takeaway; the Insight holds the depth.
- **End with a line people repeat** — "Retrieval informs. Systems decide." / "Agents plan. Systems govern."
- **Separate intelligence from control** — model proposes/predicts; system decides/owns truth.
- **No marketing language** — let the structure and reframe carry the post.
- Em dashes (`—`) are acceptable here (LinkedIn, no mermaid), but the closing principle reads sharpest with a period.

## Anti-patterns

- A different voice than the parent Insight
- Different tags than the parent Insight
- Multiple posts for one Insight, or a post with no parent Insight
- Cramming the whole Insight into the post instead of one takeaway
- Missing link back to the Insight on site
- Generic AI hype or a neutral summary instead of a sharp take

## Pre-publish checklist

```
- [ ] Voice matches the parent Insight (pov / lrn / arch / exp)
- [ ] Tags identical to the parent Insight (Core pillar + voice + topics)
- [ ] One takeaway from THE CLAIM, not a mini-essay
- [ ] Length within the voice's range
- [ ] Memorable closing line present
- [ ] Links back to the parent Insight on site
- [ ] Per-voice checklist in template.md passes
```
