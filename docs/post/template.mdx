# LinkedIn Voice Post Template

Use when **planning or drafting** a LinkedIn voice post paired to an Insight. Channel rules → [Post Engine](./Overview) · worked post → [Example](./Example).

:::info[One Insight to one post]
Same voice, same tags as the parent Insight. Link back to the Insight on site.
:::

Per-voice structure, examples, and checklists live on the four voice pages:

- [POV — Point of View](./POV)
- [LRN — Learnings](./LRN)
- [ARC — Architecture](./ARC)
- [EXP — Explain](./EXP)

---

## Planning block

```text
Parent Insight: [Title — must be published on site]
Voice: [pov / lrn / arch / exp — same as the Insight]
Core pillar: [strategy-architecture / platforms-engineering / ai-intelligence / governance-trust]
Topic tags: [same as parent Insight]
Takeaway: [one idea from THE CLAIM]
Closing line: [the memorable principle to repeat]
Insight URL: [link back to site]
Status: 💡 Idea | 📝 Draft | ✅ Ready
```

---

## Pick the voice (match the Insight)

| Voice | Core question | Length | Template page |
| --- | --- | --- | --- |
| **POV** | What belief am I challenging? | 150–400 words | [POV](./POV) |
| **LRN** | What did I learn in the field? | 200–500 words | [LRN](./LRN) |
| **ARC** | What is the architecture pattern? | 300–600 words | [ARC](./ARC) |
| **EXP** | How do I simplify without losing truth? | 200–400 words | [EXP](./EXP) |

---

## Before publishing

- [ ] **Voice** matches the parent Insight (`pov` / `lrn` / `arch` / `exp`)
- [ ] **Tags** identical to the parent Insight (Core pillar + voice + topics)
- [ ] **One takeaway** from THE CLAIM, not a mini-essay
- [ ] **Length** within the voice's range
- [ ] **Memorable closing line** present
- [ ] **Links back** to the parent Insight on site
- [ ] Per-voice checklist on [POV](./POV) · [LRN](./LRN) · [ARC](./ARC) · [EXP](./EXP) passes
