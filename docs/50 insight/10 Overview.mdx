# Insight Engine

**Insight Engine = Insights.** Every piece here is a long-form article on [jitendersharma.dev/insights](https://jitendersharma.dev/insights) — architecture-driven, 2000+ words. Pick topics from the [Content Bank](../00 content/Content-Bank#insights) · tag with [Tagging Guidelines](../00 content/Tagging-Guidelines-for-Insights).

:::info[Channel rule]
**Insight Engine → Insights on site only.** Long-form · 2000+ words · 2 per week (Tue + Thu).

**Tag every Insight:** one **Core pillar** · one **primary voice** · topic tags optional → [Tagging Guidelines](../00 content/Tagging-Guidelines-for-Insights). G.A.I.N overlays → [G.A.I.N](../10 gain/Types).
:::

:::info[After publishing]
Optional LinkedIn posts from the same research — one primary voice per post. Enrich related [G.A.I.N overlay](https://jitendersharma.dev/frameworks) or [Blueprint](https://jitendersharma.dev/blueprints) when this is an anchor Insight.
:::

## New Insight template

Use when adding to the **Insights** section of the [Content Bank](../00 content/Content-Bank#insights) — or when logging a new idea before you draft:

```text
Insight: [title]
Core pillar: [Strategy & Architecture / Platforms & Engineering / AI & Intelligence / Governance & Trust]
Primary voice: [pov / lrn / arch / exp]
Topic tags: [rag · agents · mcp · … — optional]
One-line angle: [problem or tension in one sentence]
Status: 💡 Idea
```

Framework · Blueprint · Playbook · G.A.I.N entries → add under the matching [Content Bank](../00 content/Content-Bank) section (**no tags**).

Tag slugs and examples → [Tagging Guidelines](../00 content/Tagging-Guidelines-for-Insights).

## When to use this template

- You are writing a flagship or pillar Insight
- The topic needs a full system explanation, not just a take
- Your audience is enterprise architects, platform leaders, or technical decision-makers
- You want a repeatable structure you can reuse across topics

## Structure

Follow these seven sections in order:

| # | Section | Purpose |
| --- | --- | --- |
| 1 | **Title** | Problem- or tension-driven — architectural, not marketing |
| 2 | **Context** | Why this matters now; what is broken or misunderstood |
| 3 | **System View** | Core architecture — components, flows, and responsibilities |
| 4 | **Key Design Decisions** | What matters, trade-offs, and what not to do |
| 5 | **Principles** | 3–6 sharp, repeatable design truths |
| 6 | **Enterprise Implications** | Scale, governance, risk, compliance, and operations |
| 7 | **Closing Insight** | One strong final statement |

## Template

Copy this block and replace the bracketed placeholders:

```text
Title: [Problem- or tension-driven title — clear and architectural]

1. Context
[What is happening in enterprises]
[What is broken or misunderstood]
[Why now]

2. System View
[Describe the system or reference architecture]

Flow:
[Step] → [Step] → [Step] → [Step]

What is missing (or what each layer controls):
* [Layer or gap 1]
* [Layer or gap 2]
* [Layer or gap 3]

3. Key Design Decisions
What [teams / enterprises] get wrong:
* [Mistake 1]
* [Mistake 2]

What should be done instead:
* [Decision 1]
* [Decision 2]

4. Principles
* [Principle 1]
* [Principle 2]
* [Principle 3]

5. Enterprise Implications
At enterprise scale:
* [Implication 1]
* [Implication 2]

Without [correct approach]:
* [Risk or failure mode]

6. Closing Insight
[One strong final statement — the idea you want remembered]
```

## Workflow

```text
1. Pick your topic           →  Core pillar + voice from Content Bank / 12-week plan
2. Copy the template         →  Fill in each section
3. Draft the Insight         →  Flow + layer responsibilities first
4. Tag on publish            →  domain (Core pillar) + type (voice) + topics (optional)
5. Publish to site           →  jitendersharma.dev/insights
6. Enrich overlay/blueprint  →  If anchor week — see [G.A.I.N](../10 gain/Types)
7. Optional LinkedIn post    →  One primary voice · link back to Insight
```

## Pre-publish checklist

- [ ] Title states a problem or tension — not a product pitch
- [ ] Context explains why this matters **now**, not just what the topic is
- [ ] System view includes a clear flow and named layer responsibilities
- [ ] Design decisions cover both mistakes and the correct approach
- [ ] Principles are sharp and reusable — not generic advice
- [ ] Enterprise implications address scale, governance, or compliance
- [ ] Closing insight is one memorable line, not a summary paragraph
- [ ] **One Core pillar** · **One primary voice** · **Topic tags** (optional) — [Tagging Guidelines](../00 content/Tagging-Guidelines-for-Insights)
- [ ] **G.A.I.N pillar** · overlay or blueprint enriched if anchor week — [G.A.I.N](../10 gain/Types)
