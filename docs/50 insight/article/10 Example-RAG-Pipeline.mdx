## Title

**A Production RAG Pipeline, Layer by Layer**

---

## 1. Context

Enterprise teams ship RAG by connecting an LLM to a vector database. Demos work. Production breaks on access control, stale indexes, unmeasured retrieval quality, and answers that sound authoritative but lack provenance.

This Insight maps the **full pipeline** — not just retrieval — so architects can design RAG as a system, not a integration spike.

---

## 2. System View

```text
Identity → Query → Policy filter → Retrieve → Re-rank → Context pack → Generate → Log → Feedback
```

| Layer | Responsibility |
| --- | --- |
| **Ingestion** | Source metadata, chunking, embedding, index versioning |
| **Identity & policy** | Who can ask · what corpora are in scope |
| **Retrieval** | Embed query · filter · top-k · re-rank |
| **Generation** | LLM with grounded context · citation policy |
| **Observability** | Trace retrieval set, policy decisions, output |
| **Feedback** | Eval harness · drift detection · re-embed triggers |

**What is missing in most stacks:** policy and identity sit *before* retrieval, not after generation.

---

## 3. Key Design Decisions

**What teams get wrong**

- Global vector search without document-level ACL
- One chunk size for every document type
- No offline eval set — quality judged in Slack threads

**What to do instead**

- Scope retrieval by identity and classification
- Version indexes like any data product
- Measure recall on eval sets before tuning prompts

---

## 4. Principles

- **Retrieval informs. Systems decide.** — similarity is not truth
- **Policy before prompt** — scope context before generation
- **Instrument the retrieval path** — you cannot fix what you cannot see
- **Re-rank before generate** — highest ROI quality step

---

## 5. Enterprise Implications

At enterprise scale:

- Audit asks *what was retrieved*, not just *what was said*
- Re-embedding and index rollback must be operational procedures
- Cost scales with retrieval breadth — design filters early

Without layered design:

- RAG becomes a probabilistic search UI with compliance risk

---

## 6. Closing Insight

**A production RAG pipeline is a governed data product with a language interface — not a vector database with a chat box.**
