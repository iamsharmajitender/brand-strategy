# Playbook Engine

**Playbook Engine = how-to guides on site.** Step-by-step implementation playbooks on [jitendersharma.dev/playbooks](https://jitendersharma.dev/playbooks) — from RAG and agents to observability, data pipelines, and LLM integration.

:::info[Channel rule]
**Playbook Engine → `/playbooks` on site only.** Structured paths from design decisions to production readiness.

**Not Insights** — playbooks are **untagged** · no Core pillar or voice tags · ship **after** Insights and blueprints prove the pattern.
:::

:::info[How this connects]
[Insight Engine](../50 insight/Overview) proves demand → [Blueprint Engine](../40 blueprint/Overview) shows the model → Playbook Engine tells teams **how to do it**.
:::

From the live [Architect's Handbook](https://jitendersharma.dev/playbooks):

> Practical implementation playbooks — from RAG and agents to observability, data pipelines, and LLM integration. Each guide follows a structured path from design decisions to production readiness.

## Playbook vs Insight vs Blueprint

| Channel | Job | When to ship |
| --- | --- | --- |
| **Insight** | Prove the argument | 2/week on cadence |
| **Blueprint** | Visual reference model | After anchor Insight |
| **Playbook** | Step-by-step implementation | After pattern is accepted |

**Rule:** Playbooks assume the reader already accepts the pattern — link back to Insight, blueprint, and G.A.I.N.

---

## Workflow

```text
1. Confirm pattern proven     →  Anchor Insight (+ optional Blueprint) live
2. Copy template              →  Template-Example planning block
3. Define stages              →  Numbered roadmap + key practices per stage
4. Draft playbook             →  Actionable steps · checklists · anti-patterns
5. Cross-link                 →  Insight ↔ Blueprint ↔ G.A.I.N Signature Article
6. Publish                    →  One playbook at a time when demand is clear
```
