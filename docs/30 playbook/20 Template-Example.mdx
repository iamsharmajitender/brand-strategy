# Playbook Template & Example

Use when **planning or drafting** a playbook on [/playbooks/*](https://jitendersharma.dev/playbooks). Catalog → **[Types](./Types)**.

---

## Planning block

```text
Playbook: [Title]
Route: /playbooks/[slug]
Prerequisite: [Insight + Blueprint that prove the pattern]
G.A.I.N pillar: [G / A / I / N — dominant]
Stages: [numbered implementation roadmap]
Key practices: [3–5 non-negotiables]
Status: 📋 Planned
```

---

## Page structure

Match live site style ([Build Enterprise RAG](https://jitendersharma.dev/playbooks/build-enterprise-rag)):

```text
1. Title + subtitle — outcome-focused, enterprise scope
2. Implementation roadmap — numbered stages with one-line outcomes
3. Key practices — accordion or short sections (do this · not that)
4. Related Insights · Blueprints · G.A.I.N Signature Articles
```

## Worked example — Build Enterprise RAG

From [jitendersharma.dev/playbooks/build-enterprise-rag](https://jitendersharma.dev/playbooks/build-enterprise-rag):

**Subtitle:** A step-by-step playbook to build production-ready RAG systems at scale.

**Implementation roadmap:**

| Stage | Step | Outcome |
| --- | --- | --- |
| **1** | Define the Use Case | Problem and success metrics locked |
| **2** | Data Ingestion | Collect, clean, structure data |
| **3** | Index & Retrieval | Indexes built · retrieval optimized |
| **4** | Generation Layer | LLMs integrated with grounded context |
| **5** | Evaluation & Monitoring | Quality, latency, cost measured |
| **6** | Operate & Improve | Feedback loops · continuous improvement |

**Key practices:**

- **Start with evaluation criteria** — metrics before architecture
- **Instrument from day one** — log every retrieval and generation
- **Iterate on retrieval first** — most RAG failures are retrieval problems

**Related framework:** [G.A.I.N RAG](https://jitendersharma.dev/frameworks/gain-rag) · **Blueprint:** [RAG Architecture](https://jitendersharma.dev/blueprints/rag-architecture)
