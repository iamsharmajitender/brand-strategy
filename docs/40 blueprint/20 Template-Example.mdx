# Blueprint Template & Example

Use when **planning or enriching** a blueprint on [/blueprints/*](https://jitendersharma.dev/blueprints). Catalog → **[Types](./Types)**.

---

## Planning block

```text
Blueprint: [Name]
Route: /blueprints/[slug]
Role: [one-line — what this model decides or shows]
G.A.I.N pillar: [G / A / I / N]
Anchor Insight: [title · week]
Diagram type: [layer stack / flow / decision tree]
Status: 📋 Planned
```

## Page structure

Match live site style ([Decision Model](https://jitendersharma.dev/blueprints/decision-model)):

```text
1. Title + subtitle — what this model is for
2. Section lead — why enterprises need this on one page
3. Checklist (left) + mermaid diagram (right)
4. Key patterns — accordion per pattern
5. Cross-links: anchor Insight · G.A.I.N · related playbooks
```

## Worked example — Decision Model

From [jitendersharma.dev/blueprints/decision-model](https://jitendersharma.dev/blueprints/decision-model):

**Subtitle:** Structured decision-making patterns — reasoning chains, policy gates, and human-in-the-loop workflows.

**Section lead:** AI systems make hundreds of micro-decisions per request — which tool to call, whether to escalate, how to format output. The decision model makes these choices explicit, auditable, and governable.

**Checklist:**

- Reasoning chains
- Confidence scoring
- Policy gates
- Escalation paths
- Decision audit trail

**Flow (site diagram):**

```text
Decision Request → Reasoning Chain → Confidence Score → Policy Gate
  → Automated Action OR Human Review → Audit Trail
```

**Key patterns:**

| Pattern | Purpose |
| --- | --- |
| **Reasoning Chains** | Explicit steps · inspectable agent decisions |
| **Confidence Scoring** | Low confidence triggers escalation |
| **Policy Gates** | Business rules at decision points |
| **Human-in-the-Loop** | Designed escalation — not a fallback |

**Anchor Insight (Week 6):** *Decision Model: when to use RAG vs Fine-tuning vs Agents vs MCP*
