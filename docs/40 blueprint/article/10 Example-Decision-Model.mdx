## Subtitle

Structured decision-making patterns — reasoning chains, policy gates, and human-in-the-loop workflows.

## Section lead

AI systems make hundreds of micro-decisions per request — which tool to call, whether to escalate, how to format output. The decision model makes these choices explicit, auditable, and governable.

---

## Checklist

- Reasoning chains
- Confidence scoring
- Policy gates
- Escalation paths
- Decision audit trail

---

## Decision flow

```text
Decision Request → Reasoning Chain → Confidence Score → Policy Gate
  → Automated Action OR Human Review → Audit Trail
```

---

## Pattern selection (high level)

| Need | Lean toward | Why |
| --- | --- | --- |
| Fresh enterprise knowledge, citeable answers | **RAG** | Grounded context without retraining |
| Stable domain vocabulary, repeatable style | **Fine-tuning** | When data is narrow and eval is tight |
| Multi-step workflows, tool use | **Agents** | Orchestration — with policy-governed runtime |
| Standardized tool access across systems | **MCP** | Governed tool registry — not ad-hoc integrations |

:::tip[Enterprise default]
Most teams need **RAG + better prompts** before fine-tuning. Agents require governance architecture — not just a framework choice.
:::

---

## Key patterns

| Pattern | Purpose |
| --- | --- |
| **Reasoning Chains** | Explicit steps · inspectable agent decisions |
| **Confidence Scoring** | Low confidence triggers escalation |
| **Policy Gates** | Business rules at decision points |
| **Human-in-the-Loop** | Designed escalation — not a fallback |
