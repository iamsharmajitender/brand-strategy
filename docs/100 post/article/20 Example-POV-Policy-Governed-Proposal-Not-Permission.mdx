## Post

**Proposal is not permission.**

Most enterprise agent stacks treat the LLM like an authorized actor: rules in the system prompt, tools at the end, hope in the middle.

That is not governance. That is a suggestion the model may ignore.

When an agent proposes `initiate_wire($47,500)` and money moves, regulators do not ask what the model intended. They ask:

→ Which policy version allowed it?
→ Who attested before execution?
→ Can you replay the verdict chain without chat logs?

Prompt-governed agents struggle on all three.

In a Policy-Governed Agent Runtime (PGAR):

→ The LLM **proposes** tool calls — conversation and schemas only
→ The Agentic App **holds** the token — never sent to the model
→ The Policy Engine (PDP) **decides** — deterministic verdict on structured input
→ The Policy Enforcement Point (PEP) **gates** every invocation before downstream services run

Policy in the prompt shapes behavior. It does not authorize actions.

At enterprise scale — especially in banking and regulated industries — the failure mode is not "the model misbehaved." It is "we cannot prove who authorized what, with which policy, before money moved."

**Intelligence in the LLM. Truth in the PDP.**
