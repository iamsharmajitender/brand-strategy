## Post

**"Hallucination" is not a model bug. It is a system design gap.**

Most teams respond to hallucinations by swapping models or tightening prompts.

That treats the symptom — not the architecture.

At enterprise scale, the real failure mode is this:

Similarity gets treated as truth. Retrieval gets treated as governance. And probabilistic outputs get treated as decisions.

The fix is not a better LLM.

It is a grounded system:

→ Identity-scoped context
→ Policy before generation
→ Deterministic gates where compliance requires it
→ Observability on what was retrieved — not just what was said

**Retrieval informs. Systems decide.**
