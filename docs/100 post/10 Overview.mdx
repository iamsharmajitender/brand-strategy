# Post Engine

The Post Engine is your template system for **short-form posts on LinkedIn**.

:::info[Channel rule]
**Post Engine → LinkedIn.** **Insight Engine → Insights on site.** Never swap them.
:::

:::info[Tag every post]
**Tagging:** one Core pillar · one primary voice · topic tags optional → [Tagging Guidelines for Insights](../00 content/Tagging-Guidelines-for-Insights). G.A.I.N → [G.A.I.N](../10 gain/Types).
:::

## Primary voice (type tags)

One voice per post — same slugs as site Insights:

| Slug | Label | Question | Length |
| --- | --- | --- | --- |
| `pov` | **POV** | What belief am I challenging? | 150–400 words |
| `lrn` | **LRN** | What did I learn in the field? | 200–500 words |
| `arch` | **ARC** | What is the architecture pattern or flow? | 300–600 words |
| `exp` | **EXP** | How do I explain this without losing truth? | 200–400 words |

## Workflow

```text
1. Research topic (Insight or Content Bank entry)
2. Lock tags: Core pillar + primary voice (+ topics)
3. Open matching voice template
4. Draft one post · one takeaway
5. Link back to Insight or /frameworks on site
6. Publish on LinkedIn
```

## When to use each voice

**POV** — Contrarian belief. End with a memorable principle.

**LRN** — Field lesson. What broke or changed your thinking.

**ARC** — One architecture flow or layer table — not the full Insight.

**EXP** — Simplified explanation without dumbing it down.
