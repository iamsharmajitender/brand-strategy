# Architecture Engine

**Architecture Engine = architecture notes on site.** Long-form architecture thinking published on [jitendersharma.dev/architecture](https://jitendersharma.dev/architecture) as pages take shape — distributed systems, platforms, integration, and AI-ready foundations.

:::info[Channel rule]
**Architecture Engine → `/architecture` on site.** Notes on resilience, scale, observability, event-driven design, API boundaries, AI production patterns, and governance.

**Not Insights** — architecture notes are **untagged** reference pages · no Core pillar or voice tags · no Insights backlog cadence.
:::

:::info[How this connects]
[Insight Engine](../insight/Overview) proves **one tension** then links here · Architecture Engine **holds the depth** — end-to-end systems detail, sequence flows, and contracts that single-tension Insights point to · [Blueprint Engine](../blueprint/Overview) extracts visual models · [Playbook Engine](../playbook/Overview) adds step-by-step guides.
:::

:::tip[This is a depth destination]
When an Insight raises a systems question it should not fully answer (end-to-end flow, trust boundaries, integration patterns), that depth lands here. Draft the note **in the same cycle** as the Insight that points to it.
:::

From the live [Architect's Handbook](https://jitendersharma.dev/architecture):

## Architecture vs Blueprint vs Insight

| Channel | Job | Length / form |
| --- | --- | --- |
| **Insight** | Prove one tension · tagged · cadence | Single-tension ~1000–1500 words (anchor on IP weeks) |
| **Architecture note** | Durable systems reference · untagged | Patterns + diagram + key decisions · holds Insight depth |
| **Blueprint** | One-page visual model | Layer table · flow · decision tree |

Architecture notes may **inform** blueprints (e.g. distributed systems patterns → G.A.I.N Architecture layers) but are not duplicate long-form Insights.

## Workflow

```text
1. Identify gap           →  Platform or systems topic not yet on /architecture
2. Copy template          →  [Template](./Template) planning block
3. Draft note             →  Design principles · diagram · key patterns
4. Cross-link             →  Related Insights · Blueprints · G.A.I.N
5. Publish when ready     →  No weekly cadence — ship when the note is citeable
```
