# Blueprint Engine

**Blueprint Engine = visual reference models on site.** Diagrams and layer tables on [jitendersharma.dev/blueprints](https://jitendersharma.dev/blueprints) that map G.A.I.N principles to concrete system layers — advisors and architects can cite them in one page.

:::info[Channel rule]
**Blueprint Engine → `/blueprints` on site only.** Decision trees · layer diagrams · flows — **not** long-form prose.

**Not Insights** — blueprints are **untagged** · enrich **after** an anchor Insight (or Signature Article) proves the pattern.
:::

:::info[How this connects]
[Insight Engine](../insight/Overview) proves **one tension** → Blueprint Engine extracts the **visual model** (the one-page diagram an Insight can link to instead of embedding every layer) → [Playbook Engine](../playbook/Overview) adds steps later.
:::

:::tip[This is a depth destination]
When an Insight needs a full layer diagram or decision tree, link to a blueprint here rather than packing it into the Insight. Enrich the blueprint in the same cycle as the anchor Insight that proves it.
:::

From the live [Architect's Handbook](https://jitendersharma.dev/blueprints):

> Visual blueprints that map G.A.I.N principles to concrete system layers — from agent orchestration and RAG pipelines to memory, decision-making, and the full enterprise reference architecture.

## Workflow

```text
1. Publish anchor Insight     →  Core pillar + voice + G.A.I.N letter
2. Extract diagram / flow     →  From Insight research — one page max
3. Update blueprint on site     →  Match framework page style
4. Cross-link                 →  Insight ↔ Blueprint ↔ G.A.I.N overlay
5. Optional playbook later    →  When implementation demand is clear
```
