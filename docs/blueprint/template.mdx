# Blueprint Template

Use when **planning or drafting** a blueprint on [/blueprints](https://jitendersharma.dev/blueprints). Catalog → [Types](./Types) · channel rules → [Blueprint Engine](./Overview) · worked examples → [Example](./Example).

:::info[Not an Insight]
Untagged visual reference — one page max. Enrich in the same cycle as the anchor Insight that proves it.
:::

---

## Planning block

```text
Blueprint: How to model [X]
Route: /blueprints/[slug]
Prerequisite: [Anchor Insight or Signature Article]
G.A.I.N pillar: [G / A / I / N]
Form: [layer table / decision tree / flow diagram]
Layers or decisions: [3-7 items, numbered]
Playbook (later): [/playbooks/... — optional]
Status: 💡 Idea | 📋 Planned | 📝 Draft | ✅ Live
```

---

## Page structure (published MDX)

```text
1. Title — "How to model [X]" (visual reference, not essay)
2. One-line scope — what this diagram helps decide
3. Layer table or decision tree — the one-page model
4. Flow diagram (optional) — mermaid, numbered subgraphs
5. Key decisions — 3-5 bullets architects cite
6. Related assets — Insight · G.A.I.N · Architecture · Playbook
```

**Rule:** one page. If it needs prose depth, split: Insight proves the tension, Architecture holds systems detail, Blueprint stays visual.

---

## Before publishing

- [ ] **Anchor Insight** (or Signature Article) already live or drafted this cycle
- [ ] **Untagged** — no Core pillar or voice tags
- [ ] **One page** — layer table, decision tree, or flow (not long-form prose)
- [ ] **Numbered layers or decisions** — citeable by advisors
- [ ] **No em dashes** in mermaid labels
- [ ] Cross-links to Insight · G.A.I.N · optional Playbook
- [ ] Listed in [Types catalog](./Types)
