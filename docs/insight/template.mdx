# Insight Template

Use when **planning or drafting** an Insight on [/insights](https://jitendersharma.dev/insights). Channel rules → [Insight Engine](./Overview) · tags → [Tagging Guidelines](../content-bank/tagging-guide/Tagging-Guidelines-for-Insights) · worked examples → [Example](./Example).

---

## Two tiers (pick one before drafting)

| Tier | Length | When | Job |
| --- | --- | --- | --- |
| **Single-tension** (default) | ~1000–1500 words, 1 claim, 1 diagram, 1 table | most weeks | provoke + reframe, route down |
| **Anchor** (flagship) | comprehensive, multi-diagram | IP weeks only | proof of authority |

Single-tension = **narrow scope, full depth on one claim.** Cut breadth, never rigor.

**Depth routing:** push anything beyond the one claim to its proper home and link to it.

| Content | Goes to |
| --- | --- |
| The single tension + reframe | Insight `/insights` |
| Full domain through G·A·I·N | G.A.I.N Signature `/frameworks/gain-*` |
| Sequence diagrams, contracts, systems patterns | Architecture note `/architecture` |
| One-page layer diagram / decision tree | Blueprint `/blueprints` |
| How to build / deploy / observe | Playbook `/playbooks` |

**Depth-first rule:** do not trim an Insight until its deeper sibling is live or drafted in the same cycle.

## Planning block

Copy before you draft:

```text
Insight: [Title — tension or misconception, not a product name]
Slug: [kebab-case-slug]
Tier: [single-tension (default) / anchor (IP week)]
Core pillar: [strategy-architecture / platforms-engineering / ai-intelligence / governance-trust]
Primary voice: [pov / lrn / arch / exp]
Topic tags: [rag · agents · observability · policy · … — one or more required]
THE CLAIM: [One memorable reframe in a sentence — "X is not Y"]
Routes depth to: [G.A.I.N / Architecture / Playbook / Blueprint page + link]
LinkedIn post: [same tags · planned with Insight]
Status: 💡 Idea | ✏️ Draft | ✅ Ready
```

## Voice & tone

| Do | Don't |
| --- | --- |
| Open by naming the **wrong question** everyone asks | Open with definitions or market overview |
| State **THE CLAIM** early — a contrarian reframe | Hedge with "it depends" in the opening |
| Frame as **architecture breakdown**, not opinion | Frame as thought leadership fluff |
| Use **regulated enterprise** stakes where relevant | Stay abstract or vendor-neutral only |
| Separate **intelligence (model)** from **truth / control (system)** | Blame or praise the model in isolation |
| Use **comparison tables** (wrong vs correct) | Long prose without structure |
| End with **:::tip[TAKEAWAY]::: — one line to remember** | End with a summary paragraph |

**No em dashes (`—`)** in prose, tables, admonitions, or mermaid.

## Page structure (published MDX)

### Frontmatter

```yaml
---
slug: [kebab-case]
title: [Title Case]
description: [One-line thesis — problem + architectural answer. No marketing.]
image: ./[slug].png
tags:
  - [core-pillar]      # one: strategy-architecture | platforms-engineering | ai-intelligence | governance-trust
  - [primary-voice]    # one: pov | lrn | arch | exp
  - [topic]            # one or more required
authors: [iamsharmajitender]
---
```

### Body outline — single-tension (default)

```text
# [Title — tension or misconception in plain terms]

[Hook — name the wrong conversation.]

:::tip[THE CLAIM]
**[Bold reframe.]** [2-4 sentences expanding the one claim.]
:::

<!-- truncate -->

## [The one section that proves the claim]

[One comparison table: wrong/default | correct]

[2-4 short paragraphs of mechanism. Optionally one simple diagram.]

## Where the depth lives

[Link to G.A.I.N / Architecture / Playbook / Blueprint.]

:::tip[TAKEAWAY]
**[Repeat THE CLAIM in closing form.]** [Production vs demo.]
:::
```

### Body outline — anchor (IP weeks only)

Adds on top of single-tension:

```text
## The whole system on one page
[Numbered boundaries + mermaid + bridge sentence]

## [Deep dive / walkthrough]
[Scenario + sequence diagram + JSON contracts]

## [Why architecture, not tooling]
[Table: Engineering thinks… | Architecture decides…]
```

## Before publishing

- [ ] **Tier chosen** — single-tension (default) or anchor (IP week)
- [ ] Single-tension: **one claim**, ~1000–1500 words, ≤1 diagram, ≤1 table
- [ ] Depth beyond the claim **routed** to G.A.I.N / Architecture / Playbook (+ link down)
- [ ] Deeper sibling is **live or drafted this cycle**
- [ ] **No em dashes** in prose, tables, admonitions, or mermaid
- [ ] **THE CLAIM** and **TAKEAWAY** admonitions present
- [ ] **One core pillar** + **one primary voice** + **one or more topic tags**
- [ ] **One LinkedIn voice post** planned — same tags as the Insight
- [ ] `<!-- truncate -->` after the opening block
