# Insight Engine

**Insight Engine = Insights.** Every piece here is an architecture-driven article on [jitendersharma.dev/insights](https://jitendersharma.dev/insights). Pick topics from the [Content Bank](../content-bank/Content-Bank#master-content-bank) · tag with [Tagging Guidelines](../content-bank/tagging-guide/Tagging-Guidelines-for-Insights).

:::tip[The model: keep Insights sharp, push depth down]
An Insight proves **one tension** and routes the reader to the deeper asset. The full domain model, end-to-end walkthroughs, and step-by-step depth live in **G.A.I.N**, **Architecture**, and **Playbooks** — not in every Insight.
:::

## Two tiers of Insight

| Tier | Length | Frequency | Job |
| --- | --- | --- | --- |
| **Single-tension Insight** | ~1000–1500 words · 1 claim · 1 diagram · 1 table | the default · 2/week | provoke + reframe, route down to depth |
| **Anchor Insight** | comprehensive (multi-diagram, layer-by-layer) | IP weeks only (6 · 10 · 13) | flagship proof of authority |

**Single-tension is not "short and shallow."** It is **narrow scope, full depth on one claim** — cut breadth, never rigor. Reserve the comprehensive style for anchor weeks so the flagship pieces stay special.

## Where the depth goes

Prove the one tension in the Insight, then link out:

| Content | Lives in | Route |
| --- | --- | --- |
| The single tension + the reframe | **Insight** | `/insights` |
| Full domain model, layer-by-layer through G·A·I·N | **[G.A.I.N Signature Article](../gain/Overview)** | `/frameworks/gain-*` |
| End-to-end sequence, contracts, systems patterns | **[Architecture note](../architecture/Overview)** | `/architecture` |
| One-page layer diagram / decision tree | **[Blueprint](../blueprint/Overview)** | `/blueprints` |
| How to build / deploy / observe | **[Playbook](../playbook/Overview)** | `/playbooks` |

:::important[Depth-first rule]
Never trim an Insight's depth until its deeper sibling (G.A.I.N / Architecture / Playbook / Blueprint) is **live or drafted in the same cycle**. A trimmed Insight that links to nothing reads as bait. Every single-tension Insight ends with a **link down** to the asset that carries the detail.
:::

:::info[Channel rule]
**Insight Engine → Insights on site only.** 2 per week (Tue + Thu) · default single-tension · anchor on IP weeks.

**Tag every Insight:** one **Core pillar** · one **primary voice** · **one or more topic tags** → [Tagging Guidelines](../content-bank/tagging-guide/Tagging-Guidelines-for-Insights) · [Core Pillars](../content-bank/tagging-guide/core-pillars) · [Voice Tags](../content-bank/tagging-guide/voice-tags) · [Topic Tags](../content-bank/tagging-guide/topic-tags). G.A.I.N overlays → [G.A.I.N](../gain/Types).
:::

:::info[After publishing]
Publish **one LinkedIn voice post per Insight** — same Core pillar, voice, and topic tags; shorter format; link back to the Insight. Enrich related [G.A.I.N overlay](https://jitendersharma.dev/frameworks) or [Blueprint](https://jitendersharma.dev/blueprints) when this is an anchor Insight.
:::

## Structure — single-tension Insight (default)

Keep it to the one claim. One supporting section, one diagram or table, then route the reader down.

| # | Section | Purpose |
| --- | --- | --- |
| 1 | **Title** | The tension or misconception in plain terms |
| 2 | **Hook** | Name the wrong question; state what actually decides the outcome |
| 3 | **THE CLAIM** | One quotable reframe — `X is not Y` |
| 4 | **The one section** | One comparison table + short mechanism that proves the claim |
| 5 | **Where the depth lives** | One line + a link down to G.A.I.N / Architecture / Playbook |
| 6 | **TAKEAWAY** | Restate the claim; production vs demo |

```text
Title: [The tension in plain terms]

Hook
[Wrong question / common belief.]
[Why it fails. What actually decides the outcome.]

THE CLAIM
[X is not Y. It is <architectural reality>.]

The one section that proves it
[Comparison table: wrong/default | correct]
[2-4 short paragraphs of mechanism. One simple diagram, optional.]

Where the depth lives
[This Insight makes the case. The full model / walkthrough / steps live in →]
[Link: G.A.I.N <Domain> / Architecture note / Playbook]

TAKEAWAY
[Restate the claim in closing form.]
```

## Structure — anchor Insight (IP weeks only)

For flagship weeks (e.g. 12-week plan weeks 6, 10, 13), keep the single-tension skeleton and **add** the comprehensive layers:

- **The whole system on one page** — numbered boundaries/stages + mermaid + bridge sentence
- **Deep dive / walkthrough** — concrete scenario + sequence diagram + JSON contracts
- **Why architecture, not tooling** — "Engineering thinks… / Architecture decides…" table
- **Enterprise stakes** — auditors, regulators, compliance, operational cost

## Workflow

```text
1. Pick the tier            →  Single-tension (default) or anchor (IP week)
2. Pick your topic          →  Core pillar + voice from Content Bank / 12-week plan
3. Draft the claim first    →  Hook → THE CLAIM → one proving section
4. Route the depth          →  Move anything beyond the claim to G.A.I.N / Architecture / Playbook
5. Link down                →  End with a link to the deeper asset (must exist or be drafted)
6. Tag on publish           →  domain (Core pillar) + type (voice) + topics (one or more)
7. Publish to site          →  jitendersharma.dev/insights
8. LinkedIn voice post      →  One per Insight · same tags · link back · [Post Engine](../post/Overview)
```

## Pre-publish checklist

- [ ] **Tier chosen** — single-tension (default) or anchor (IP week)
- [ ] Single-tension: **one claim**, ~1000–1500 words, ≤1 diagram, ≤1 table
- [ ] Depth beyond the claim **routed** to G.A.I.N / Architecture / Playbook (+ link down)
- [ ] Deeper sibling is **live or drafted this cycle** (depth-first rule)
- [ ] Title states a problem or tension — not a product pitch
- [ ] **THE CLAIM** and **TAKEAWAY** present
- [ ] One comparison table that proves the claim
- [ ] **One Core pillar** · **One primary voice** · **One or more topic tags** — [Core Pillars](../content-bank/tagging-guide/core-pillars) · [Voice Tags](../content-bank/tagging-guide/voice-tags) · [Topic Tags](../content-bank/tagging-guide/topic-tags)
- [ ] **One LinkedIn voice post** planned — same tags as the Insight
- [ ] **G.A.I.N pillar** · overlay or blueprint enriched if anchor week — [G.A.I.N](../gain/Types)
