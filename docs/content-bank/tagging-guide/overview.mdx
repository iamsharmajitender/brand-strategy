# Tagging Guidelines

How every asset type gets tagged before you ship. **Insights** carry voice and LinkedIn pairing. **G.A.I.N · Blueprint · Architecture · Playbook** carry **Core pillar + topic tags** only.

**Backlog:** [Content Bank](../Content-Bank) · [Master content bank](../Content-Bank#master-content-bank)

---

## At a glance

| Asset type | Route | Core pillar | Primary voice | Topic tags | LinkedIn | Rule |
| --- | --- | --- | --- | --- | --- | --- |
| **Insight** | `/insights` | **One** (required) | **One** (required) | **One or more** (required) | **One per Insight** (same tags) | Proof · cadence content |
| **LinkedIn voice post** | LinkedIn | Same as Insight | Same as Insight | Same as Insight | — | Not a separate backlog |
| **G.A.I.N** | `/frameworks/gain-*` | **One** (required) | — | **One or more** (required) | — | Optional in depth stack |
| **Blueprint** | `/blueprints` | **One** (required) | — | **One or more** (required) | — | How to model |
| **Architecture** | `/architecture` | **One** (required) | — | **One or more** (required) | — | How to architect |
| **Playbook** | `/playbooks` | **One** (required) | — | **One or more** (required) | — | How to build |

:::tip[The rules in one line]
**Insight:** one Core pillar · one voice · one or more topics · one LinkedIn post.

**G.A.I.N · Blueprint · Architecture · Playbook:** one Core pillar · one or more topics · **no voice tag**.
:::

:::info[Inherit from the topic stack]
Depth assets share tags with their [stack topic row](../Content-Bank#master-content-bank). When an Insight proves *RAG is Not a Database* under **AI & Intelligence** with topic `rag`, **G.A.I.N RAG**, **RAG Architecture**, **enterprise RAG architecture note**, and **Build Enterprise RAG** all use the same Core pillar and `rag` topic unless a second topic clearly applies.
:::

## Insights

Insights are the only asset type with a **primary voice** and a **LinkedIn pair**.

### What to tag

| Tag | Rule | Values |
| --- | --- | --- |
| **Core pillar** | Exactly one | [Core pillar catalog](./core-pillars) |
| **Primary voice** | Exactly one | [Voice catalog](./voice-tags) |
| **Topic tags** | One or more | [Topic catalog](./topic-tags) |
| **LinkedIn post** | One per Insight | Same Core pillar, voice, and topics |

Voice is how the **Insight reads** on site. The paired **LinkedIn post uses the same voice and the same tags**. Templates → [Voice Tags](./voice-tags).

### LinkedIn (one per Insight)

- Same **Core pillar**, **primary voice**, and **topic tags** as the parent Insight
- Shorter format · always **link back** to the Insight on site
- Plan when you plan the Insight · publish after the Insight goes live

### Site frontmatter (Insights)

| Planning tag | Site frontmatter |
| --- | --- |
| Core pillar | `domain` tag (slug) |
| Primary voice | `type` tag (slug) |
| Topic tags | `topic` tags (slugs) |

## G.A.I.N · Blueprint · Architecture · Playbook

Depth assets in the [content stack](../Content-Bank#how-each-insight-stacks). **Not Insights** — no voice tag, no LinkedIn.

### What to tag

| Tag | Rule |
| --- | --- |
| **Core pillar** | Exactly one — match the parent topic stack or anchor Insight |
| **Topic tags** | One or more — match the topic domain (e.g. `rag`, `agents`) |
| **Primary voice** | **Do not tag** |
| **LinkedIn** | **Do not create** — only Insights get LinkedIn posts |

### G.A.I.N planning note

Signature Articles also carry a **G.A.I.N pillar letter** (`G` · `A` · `I` · `N`) for framework planning. That is **not** a Core pillar and **not** a site voice tag. Catalog → [G.A.I.N Types](../../gain/Types).

### Site frontmatter (depth assets)

| Planning tag | Site frontmatter |
| --- | --- |
| Core pillar | `domain` tag (slug) |
| Topic tags | `topic` tags (slugs) |

## Pre-planning checklist

### Insight

- [ ] **One Core pillar** — [catalog](./core-pillars)
- [ ] **One primary voice** — [catalog](./voice-tags)
- [ ] **One or more topic tags** — [catalog](./topic-tags)
- [ ] **One LinkedIn voice post** planned (same tags)
- [ ] Listed in [Content Bank](../Content-Bank) with **Stack topic** when applicable

### G.A.I.N · Blueprint · Architecture · Playbook

- [ ] **One Core pillar** (inherit from topic stack or anchor Insight)
- [ ] **One or more topic tags**
- [ ] **No voice tag**
- [ ] Row updated in [Master content bank](../Content-Bank#master-content-bank)

## Weekly workflow

```text
Monday     → Lock Tue + Thu Insights · Core pillar + voice + topics
Tuesday    → Publish Insight · tag on site · LinkedIn post next day
Wednesday  → Publish LinkedIn voice post (same tags · link back)
Thursday   → Publish second Insight · tag on site
Friday     → LinkedIn for Thu Insight · tag depth assets in stack (Core pillar + topics)
```
