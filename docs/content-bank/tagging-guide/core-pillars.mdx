# Core Pillars

Four domains only — same as [jitendersharma.dev](https://jitendersharma.dev). **Not** LLM, RAG, or Cloud as pillars; those are **[topic tags](./topic-tags)**.

**Rule:** exactly **one Core pillar** on every tagged asset — Insight, G.A.I.N, Blueprint, Architecture note, or Playbook.

**Tagging rules:** [Tagging Guidelines](./Tagging-Guidelines-for-Insights) · **Master list:** [Content Bank](../Content-Bank#master-content-bank)

| Icon | Slug | Core pillar | Definition |
| --- | --- | --- | --- |
| ⬡ | `strategy-architecture` | [**Strategy & Architecture**](https://jitendersharma.dev/insights/tags/strategy-architecture) | Roadmaps, reference models, and design authority for enterprise transformation. |
| ⚙ | `platforms-engineering` | [**Platforms & Engineering**](https://jitendersharma.dev/insights/tags/platforms-engineering) | Cloud-native, event-driven, and observable systems that scale. |
| ◈ | `ai-intelligence` | [**AI & Intelligence**](https://jitendersharma.dev/insights/tags/ai-intelligence) | Governed agents, RAG, and production AI on enterprise foundations. |
| ⛊ | `governance-trust` | [**Governance & Trust**](https://jitendersharma.dev/insights/tags/governance-trust) | Policy, compliance, and operational resilience in regulated environments. |

**Planning tables** may use the full name (e.g. AI & Intelligence). **Site frontmatter** uses the slug as the `domain` tag.

**Not Core pillars:** topic tags (`rag`, `agents`, …) → [Topic Tags](./topic-tags) · voice tags (`pov`, `lrn`, `arch`, `exp`) → [Voice Tags](./voice-tags) · G.A.I.N pillar letter (`G` · `A` · `I` · `N`) for framework planning → [G.A.I.N Types](../../gain/Types).
