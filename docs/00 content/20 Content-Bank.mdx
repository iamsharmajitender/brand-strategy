# Content Bank

Master list for [jitendersharma.dev](https://jitendersharma.dev). Pull **2 Insights per week** (Tue + Thu) into the [12 Weeks Plan](./Weeks-Plan).

**Status key:** ✅ Published · 🟧 In progress · 📝 Draft · 📋 Planned · 💡 Idea

:::tip[What gets tagged]
**Insights only** — each long-form article gets **one Core pillar · one primary voice · topic tags (optional)** → [Tagging Guidelines for Insights](./Tagging-Guidelines-for-Insights).

**LinkedIn voice posts** — optional extracts from Insight research · same tag taxonomy for planning · **not** a separate backlog channel.

**G.A.I.N · Blueprints · Playbooks · Architecture** — listed below for planning · **no Core pillar or voice tags**.
:::

## Content types

Six asset types across [jitendersharma.dev](https://jitendersharma.dev). Each has a different job — do not mix channels (e.g. a Signature Article is **not** an Insight).

| Type | Route | What it does | Tagged? | When to ship | Engine |
| --- | --- | --- | --- | --- | --- |
| **Insight** | `/insights` | Long-form **proof** — one argument per piece. Proves a tension, architecture pattern, or field lesson in 2000+ words. Drives your publishing cadence and research. | **Yes** — Core pillar · primary voice · optional topics | **2/week** (Tue + Thu) | [Insight Engine](../50 insight/Overview) |
| **LinkedIn voice post** | LinkedIn | Short **extract** from Insight research — one takeaway, one voice (POV · LRN · ARC · EXP). Links back to the parent Insight on site. | Same taxonomy for planning only | Optional, after Insight | [Post Engine](../100 post/Overview) |
| **G.A.I.N** | `/frameworks` | **Operating model + domain reference.** Core framework (G · A · I · N loop) plus Signature Articles — each maps a whole domain (LLM · RAG · Agents · …) **through G.A.I.N**. Permanent IP, not weekly content. | **No** — G.A.I.N pillar letter only | One Signature Article at a time · enrich after anchor Insight | [Gain Engine](../10 gain/Overview) |
| **Blueprint** | `/blueprints` | **Visual reference model** — one-page diagram advisors cite. Decision trees, layer tables, agent flows, RAG pipelines. Extracts the pattern from an Insight into a durable visual. | **No** | After anchor Insight proves the pattern · IP weeks (6 · 10 · 13) | [Blueprint Engine](../40 blueprint/Overview) |
| **Architecture** | `/architecture` | **Durable systems notes** — platform and engineering reference. Distributed systems, event-driven design, API boundaries, AI production patterns. Holds depth Insights don't repeat. | **No** | When the note is citeable — no weekly cadence | [Architecture Engine](../20 architecture/Overview) |
| **Playbook** | `/playbooks` | **Step-by-step implementation** — checklists and runbooks. Assumes the reader already accepts the pattern. Tells teams *how* to build, deploy, or observe. | **No** | Last in the stack — after Insight + optional Blueprint | [Playbook Engine](../30 playbook/Overview) |

### How each type fits

**Insight** is the engine. You research once, tag it, publish on site, and everything else compounds from it.

**LinkedIn voice post** is optional and Insight-only for now. Same research, different voice, shorter format — always link back to the Insight.

**G.A.I.N** is your brand spine. Insights prove a point; Signature Articles turn that proof into permanent framework IP a client can navigate.

**Blueprint** distills an Insight into a visual model — the diagram you put in a deck or advisory conversation.

**Architecture** holds platform and systems depth that isn't a tagged essay — reference notes that age well and support multiple Insights.

**Playbook** ships when demand is proven — the "how to implement" layer after the argument (Insight) and the model (Blueprint) are live.

### How they stack

```text
Insight (tagged · /insights)
    ↓ optional
LinkedIn voice post (POV · LRN · ARC · EXP)
    ↓ enrich when anchor week
G.A.I.N Signature Article · Blueprint · Architecture note · Playbook
```
