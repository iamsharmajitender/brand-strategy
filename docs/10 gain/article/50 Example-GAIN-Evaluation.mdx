## One-line angle

Evaluation is a production gate — not a notebook you run once before launch.

## Opening thesis

Teams treat eval as a pre-launch checklist. G.A.I.N Evaluation treats it as a **continuous adaptive layer**: offline harnesses, online signals, rollout gates, and rollback triggers — tied to every G.A.I.N pillar, not just answer quality.

---

## Why this domain needs G.A.I.N

| Pillar | What you evaluate |
| --- | --- |
| **G** | Retrieval recall · grounding fidelity · citation accuracy |
| **A** | Drift · cohort quality · post-rollout regression |
| **I** | Plan correctness · tool selection · escalation appropriateness |
| **N** | Latency · cost · error budgets under load |

Without **A**, G and I improvements are guesses. This article defines the eval **system**, not a single metric.

---

## Grounded (G)

- Golden datasets per use case — owned by domain experts, versioned
- Grounding checks: answer supported by retrieved chunks? citations match?
- Safety evals: policy violations, PII leakage, hallucination rate on held-out sets

---

## Adaptive (A) — dominant

**Eval lifecycle**

```text
Define metrics → Offline harness → Shadow deploy → Cohort rollout → Monitor → Rollback or promote
```

**Patterns**

- **Quality gates** — no promotion without passing threshold on *all* pillar-relevant suites
- **Human review queues** — sample production traffic for label feedback
- **Regression bank** — every incident becomes a permanent eval case

**Decisions**

- Separate evals for retrieval vs generation — most failures are retrieval
- Eval model version independently from serving model version

**Anti-patterns**

- One BLEU score for enterprise RAG
- Eval only in dev — production drift invisible

---

## Intelligent (I)

- Agent evals: did the plan use the right tools? was escalation correct?
- Policy gate tests — automated suites for deny/allow paths
- Counterfactual replay: "what if we had routed to model B?"

---

## Native (N)

- Eval runs at CI/CD, pre-deploy, and scheduled production sampling
- Cost of eval tracked — cheap eval models for triage, expensive for final gate
- Eval infrastructure scales with traffic — not a bottleneck on release
