# Gain Engine

**Gain Engine = G.A.I.N on site.** The operating model and domain Signature Articles live on [jitendersharma.dev/frameworks](https://jitendersharma.dev/frameworks) — not in the Insights backlog.

:::info[Channel rule]
**Gain Engine → Frameworks on site only.** Core framework at `/frameworks` · domain articles at `/frameworks/gain-*`.

**Not Insights** — Signature Articles carry **no** Core pillar tag · **no** primary voice tag · map a whole domain through **G · A · I · N**.
:::

:::info[How this connects]
Research in [Insight Engine](../50 insight/Overview) → permanent IP here → optional [Blueprint](../40 blueprint/Overview) or [Playbook](../30 playbook/Overview) extraction later.
:::

**G.A.I.N** = **G**overned **A**I-**N**ative Systems — the **one proprietary idea** people associate with this brand. Every Insight, Signature Article, and advisory conversation should reinforce governed, enterprise-scale AI architecture.

```text
Data & Context → G → A → I → N → Feedback Loop → Data & Context
```

| Letter | Pillar | One line |
| --- | --- | --- |
| **G** | Grounded | Truth, context, and knowledge alignment |
| **A** | Adaptive | Learning, feedback, and continuous evolution |
| **I** | Intelligent | Reasoning, agents, and decision systems |
| **N** | Native | Scalable, modular, future-ready design |

Full pillar definitions · Signature Article catalog · build track → **[Types](./Types)**

## Gain Engine vs other channels

| Channel | Job | Example |
| --- | --- | --- |
| **Insight** | Prove one argument · tagged · 2/week | *RAG is Not a Database* |
| **G.A.I.N Signature Article** | Map a domain through G · A · I · N · permanent IP | **G.A.I.N RAG** — full enterprise RAG reference |
| **Blueprint** | Visual model on one page | [RAG Architecture](https://jitendersharma.dev/blueprints/rag-architecture) |
| **Playbook** | Step-by-step implementation | [Build Enterprise RAG](https://jitendersharma.dev/playbooks/build-enterprise-rag) |

An Insight **can** be G.A.I.N-aligned. It is **not** a Signature Article until you publish on `/frameworks/gain-*`.

## Workflow

```text
1. Pick domain              →  One Signature Article from Types catalog
2. Copy template            →  Template-Example planning block
3. Draft G · A · I · N      →  Map patterns to each pillar section
4. Publish one at a time    →  Ship G.A.I.N LLM before RAG or Agents
5. Cross-link               →  Signature Article ↔ related Insights ↔ core framework
6. Enrich downstream        →  Blueprint or Playbook when pattern is proven
```

**Publish one domain at a time.** Schedule → **[12 Weeks Plan](../00 content/100 Twelve-Weeks-Plan)**
